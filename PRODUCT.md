# Product

## Register

product

## Users

Cortex is for people who write, organize, and maintain Markdown notes in a desktop workspace.
They are usually in a keyboard-heavy editing flow and need formatting, navigation, settings,
plugins, and sync to stay close to the note without interrupting concentration.

## Product Purpose

Cortex exists to make Markdown editing feel native, extensible, and dependable across a modular
desktop platform. Success means users can create, structure, search, format, and sync notes with
confidence while the app preserves plain Markdown as the durable source of truth.

## Brand Personality

Quiet, precise, capable. The interface should feel like a serious writing tool with enough warmth
to invite daily use, but it should never compete with the note content or turn routine actions into
spectacle.

## Anti-references

Avoid marketing-page composition inside the product, decorative gradients, oversized cards,
novel controls where standard app patterns work better, and any design that makes Markdown feel
like a hidden implementation detail rather than the user's own material.

## Design Principles

- Keep the editor in charge: supporting UI should appear only when it helps the current writing
  gesture and should disappear without disturbing cursor behavior.
- Prefer earned familiarity: command palettes, menus, settings rows, and editor affordances should
  feel recognizable to users of polished desktop tools.
- Preserve plain Markdown: formatting helpers may accelerate syntax entry, but they should not hide
  or mutate the document in surprising ways.
- Design for density with calm: compact controls, restrained color, and predictable spacing should
  make repeated use feel fast rather than cramped.
- Let extensions fit the host: plugin and theme surfaces should reuse Cortex conventions instead
  of creating competing interface vocabularies.

## Accessibility & Inclusion

Default to keyboard-first workflows, visible focus states, reduced-motion-safe transitions, and
WCAG AA contrast for text and essential controls. Editor enhancements must not trap focus or break
screen-reader and cursor expectations from CodeMirror.
